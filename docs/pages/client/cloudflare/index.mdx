# Cloudflare Workers

Cloudflare Workers has built-in observability that can export traces via OTLP. No SDK or code changes are needed in your Worker — just configure a destination in the Cloudflare dashboard and enable it in your `wrangler.jsonc`.

## Step 1: Create a destination in the Cloudflare dashboard

Navigate to **Compute (Workers) → Observability → Destinations**, then click **"+ Add Destination"**.

![Cloudflare Observability Destinations](/cloudflare-destinations.png)

Fill in the following fields:

- **Destination Type**: Traces
- **OTLP Traces Endpoint**: `https://cloud.tracewayapp.com/api/otel/v1/traces` (or your self-hosted equivalent)
- **Destination Name**: `traceway`
- **Custom Header**: `Authorization` = `Bearer <project_token>`

Replace `<project_token>` with your project token from the Traceway dashboard under **Project Settings**.

![Add Destination form](/cloudflare-add-destination.png)

## Step 2: Enable in wrangler.jsonc

Add the `observability.traces` configuration to your `wrangler.jsonc`, referencing the destination name you created above:

```jsonc
{
  "observability": {
    "traces": {
      "enabled": true,
      "destinations": ["traceway"],
      "head_sampling_rate": 1
    }
  }
}
```

Set `head_sampling_rate` to a value between `0` and `1` to control what fraction of requests are traced. Use `1` to trace every request.

## Step 3: Deploy

Deploy your Worker:

```bash
npx wrangler deploy
```

Traces will appear in Traceway automatically.
