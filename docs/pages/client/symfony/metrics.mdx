# Metrics

The OTel auto-instrumentation captures basic request metrics automatically. To track custom application-specific metrics, use the OpenTelemetry PHP API to create counters, gauges, and histograms.

## Creating a Meter

Get a meter instance from the global meter provider:

```php
use OpenTelemetry\API\Globals;

$meter = Globals::meterProvider()->getMeter('app');
```

## Counter

Counters track cumulative values that only go up (e.g., total orders, requests processed):

```php
$meter = Globals::meterProvider()->getMeter('app');
$orderCounter = $meter->createCounter('orders.created', 'orders', 'Total orders created');

// Increment by 1
$orderCounter->add(1);

// Increment with attributes
$orderCounter->add(1, ['plan' => 'pro', 'region' => 'eu']);
```

## Histogram

Histograms track distributions of values (e.g., response times, payload sizes):

```php
$meter = Globals::meterProvider()->getMeter('app');
$durationHistogram = $meter->createHistogram('order.processing_ms', 'ms', 'Order processing time');

$start = hrtime(true);
$this->processOrder($order);
$durationMs = (hrtime(true) - $start) / 1e6;

$durationHistogram->record($durationMs);
```

Traceway converts histograms into average and count metrics (see [OTel metrics](/client/otel/metrics#histogram-handling)).

## Gauge

Gauges track point-in-time values that can go up or down (e.g., queue depth, active connections):

```php
$meter = Globals::meterProvider()->getMeter('app');

// Observable gauge — value is read by a callback at export time
$meter->createObservableGauge('queue.size', 'items', 'Current queue depth')
    ->observe(function ($observer) {
        $observer->observe($this->queue->count());
    });
```

## Use Cases

### Business Metrics

```php
$meter = Globals::meterProvider()->getMeter('app');
$revenueCounter = $meter->createCounter('payments.revenue', 'usd', 'Total revenue');
$signupCounter = $meter->createCounter('users.signups', 'users', 'Total signups');

// In your payment service
public function processPayment(Order $order): void
{
    $this->gateway->charge($order->total);
    $revenueCounter->add($order->total, ['plan' => $order->plan]);
}

// In your registration controller
public function register(Request $request): Response
{
    $user = $this->userService->create($request);
    $signupCounter->add(1);
    return new JsonResponse($user, 201);
}
```

### Performance Metrics

```php
$meter = Globals::meterProvider()->getMeter('app');
$apiLatency = $meter->createHistogram('external_api.latency_ms', 'ms', 'External API call duration');

public function fetchExternalData(string $endpoint): array
{
    $start = hrtime(true);
    try {
        return $this->client->request('GET', $endpoint)->toArray();
    } finally {
        $durationMs = (hrtime(true) - $start) / 1e6;
        $apiLatency->record($durationMs, ['endpoint' => $endpoint]);
    }
}
```

### Resource Metrics

```php
$meter = Globals::meterProvider()->getMeter('app');

$meter->createObservableGauge('cache.entries', 'items', 'Cache entry count')
    ->observe(function ($observer) {
        $observer->observe($this->cache->getStats()['entries']);
    });

$meter->createObservableGauge('cache.memory_mb', 'MB', 'Cache memory usage')
    ->observe(function ($observer) {
        $observer->observe($this->cache->getStats()['memory'] / 1024 / 1024);
    });
```

## Metric Naming Conventions

Use dot-separated names for organization:

```php
// Good naming
$meter->createCounter('orders.created');
$meter->createHistogram('db.query_ms');
$meter->createCounter('cache.hits');

// Bad naming
$meter->createCounter('created');         // Too vague
$meter->createCounter('orderCount');     // Inconsistent style
```

## Registering Metrics as a Service

Define your meters in a Symfony service for dependency injection:

```php
// src/Telemetry/AppMetrics.php
namespace App\Telemetry;

use OpenTelemetry\API\Globals;
use OpenTelemetry\API\Metrics\CounterInterface;
use OpenTelemetry\API\Metrics\HistogramInterface;

class AppMetrics
{
    public readonly CounterInterface $ordersCreated;
    public readonly HistogramInterface $orderProcessingMs;
    public readonly CounterInterface $paymentSuccess;
    public readonly CounterInterface $paymentFailed;

    public function __construct()
    {
        $meter = Globals::meterProvider()->getMeter('app');

        $this->ordersCreated = $meter->createCounter('orders.created', 'orders');
        $this->orderProcessingMs = $meter->createHistogram('orders.processing_ms', 'ms');
        $this->paymentSuccess = $meter->createCounter('payments.success', 'payments');
        $this->paymentFailed = $meter->createCounter('payments.failed', 'payments');
    }
}
```

Then inject it where needed:

```php
class OrderService
{
    public function __construct(
        private readonly AppMetrics $metrics,
    ) {}

    public function createOrder(array $items): Order
    {
        $start = hrtime(true);
        try {
            $order = $this->doCreateOrder($items);
            $this->metrics->ordersCreated->add(1);
            return $order;
        } finally {
            $this->metrics->orderProcessingMs->record((hrtime(true) - $start) / 1e6);
        }
    }
}
```
