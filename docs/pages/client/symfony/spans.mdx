# Spans

The bundle creates spans for HTTP requests automatically. To measure sub-operations like database queries, API calls, or business logic, create spans using the bundle's `TracingInterface` helper or the OpenTelemetry PHP API directly.

## Using TracingInterface

The bundle provides a `TracingInterface` service you can inject into any service or controller. It handles span creation, activation, error recording, and cleanup:

```php
use Traceway\OpenTelemetryBundle\TracingInterface;

class OrderService
{
    public function __construct(
        private readonly TracingInterface $tracing,
    ) {}

    public function processOrder(int $orderId): void
    {
        $this->tracing->trace('order.validate', function () use ($orderId) {
            $this->validateOrder($orderId);
        });

        $this->tracing->trace('order.charge', function () use ($orderId) {
            $this->chargePayment($orderId);
        });
    }
}
```

### With Attributes and Span Kind

```php
use OpenTelemetry\API\Trace\SpanKind;

$result = $this->tracing->trace(
    'stripe.charge',
    fn () => $this->stripe->charge($amount),
    attributes: ['payment.amount' => $amount, 'payment.currency' => 'usd'],
    kind: SpanKind::KIND_CLIENT,
);
```

### Nested Spans

Child spans are automatically linked to the active parent span:

```php
$this->tracing->trace('order.fulfill', function () {
    $this->tracing->trace('inventory.reserve', fn () => $this->reserve());
    $this->tracing->trace('payment.charge', fn () => $this->charge());
    $this->tracing->trace('email.send', fn () => $this->notify());
});
```

## Using the OpenTelemetry API Directly

For advanced use cases (span events, manual status codes, lazy attributes), use the tracer directly:

```php
use OpenTelemetry\API\Globals;

$tracer = Globals::tracerProvider()->getTracer('app');

$span = $tracer->spanBuilder('process-order')->startSpan();
$scope = $span->activate();
try {
    $this->processOrder($orderId);
} finally {
    $scope->detach();
    $span->end();
}
```

Always end spans in a `finally` block to ensure they are closed even if an exception is thrown.

## Adding Attributes

Attach metadata to spans for filtering and debugging:

```php
$span = $tracer->spanBuilder('db-query')->startSpan();
$span->setAttribute('db.system', 'mysql');
$span->setAttribute('db.statement', 'SELECT * FROM users WHERE id = ?');
$span->setAttribute('db.row_count', $rowCount);
$span->end();
```

## Recording Errors on Spans

When a span's operation fails, record the exception and set the status:

```php
use OpenTelemetry\API\Trace\StatusCode;

$span = $tracer->spanBuilder('external-api-call')->startSpan();
try {
    $response = $this->httpClient->request('GET', 'https://api.example.com/data');
} catch (\Throwable $e) {
    $span->recordException($e);
    $span->setStatus(StatusCode::STATUS_ERROR, $e->getMessage());
    throw $e;
} finally {
    $span->end();
}
```

## Span Naming Conventions

Use descriptive names that indicate the operation type:

| Good | Bad |
|------|-----|
| `db.users.find` | `query` |
| `cache.sessions.get` | `cache` |
| `stripe.charge` | `api` |
| `s3.upload-image` | `upload` |
| `email.send-welcome` | `send` |
